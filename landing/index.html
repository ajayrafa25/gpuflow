One command to submit. One dashboard to monitor.

No config files. No YAML. Just your image and your training command.

Submit a job CLI
# run any command in any image gpuflow run \ --image pytorch/pytorch:2.1.0 \ --gpus 2 \ --user alice \ torchrun train.py
Navigate first, then run CLI
# code lives on a shared drive gpuflow run \ --image myrepo/env:v3 \ --user bob \ "cd /workspace/bob/proj && torchrun train.py"
Check job queue CLI
# see who's running what gpuflow status   # stream logs live gpuflow logs <job-id> -f   # cancel a job gpuflow cancel <job-id>
Experiment tracking MLflow built-in
# MLflow URI auto-injected # into every container   # in your train.py: import mlflow mlflow.log_metric("loss", loss) mlflow.log_param("lr", lr) # → shows in dashboard

How teams use it

From first install to running experiments in minutes.

Step 01 — Admin

Install & start the server

One command on the GPU machine. MLflow and the dashboard start automatically.

pip install gpuflow gpuflow-server start
Step 02 — Admin

Pull Docker images

Any image on the server becomes available in the dashboard dropdown for debug sessions.

docker pull pytorch/pytorch:2.1.0 docker pull myrepo/custom-env:v3
Step 03 — Developer

Debug, then launch

Open VS Code from the dashboard to iterate. When ready, submit the job with one command.

gpuflow run \ --image myrepo/custom-env:v3 \ --gpus 2 --user alice \ torchrun train.py
